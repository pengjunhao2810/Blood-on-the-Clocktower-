## 三、AI 学习机制正确性

### 机制验证方法

| 方法 | 说明 |
|------|------|
| 自对弈测试 | 每轮改动后运行3-50局验证无崩溃 |
| 规则硬约束 | 16项代码级拦截，不由AI自行判断 |
| 过程奖励校准 | 18项奖励信号，通过22000+局收敛到52% |
| 数据交叉验证 | 导出Word文档逐条核对角色/私聊/夜晚信息 |

### 学习算法

| 算法 | 用途 | 说明 |
|------|------|------|
| **REINFORCE** | 邪恶方策略训练 | 策略梯度法，优化杀目标/提名/投票 |
| **过程奖励** | 中间行为指导 | 刀信息位+0.3、反推好人-0.4等18项信号 |
| **MLP网络** | 状态→动作映射 | 10维×128隐层×3输出头(kill/nom/vote) |
| **模板检索** | 发言生成 | 48类对话模板+used集合防重复 |
| **启发式规则** | 好人决策 | 对跳优先、空泛过滤、信息交叉权重 |

### 模型架构

```
Observation(10维/玩家: alive/is_self/team/role_type/claim/claim_cat/suspicion/trust/is_demon/is_minion)
    ↓
MLP(128隐层, ReLU)
    ↓
├── kill_head:  softmax → P(kill each player)
├── nom_head:   softmax → P(nominate each player)
└── vote_head:  sigmoid  → P(vote yes)
```

### 探索策略

| 阶段 | epsilon | 说明 |
|------|---------|------|
| 初始 | 0.6 | 高探索率 |
| 中期 | 0.3~0.5 | 逐步收敛 |
| 最终 | 0.05 | 低探索 |

## 七、推理能力正确性

### 好人推理链

| 公理 | 实现 |
|------|------|
| 查验为好人→暂排入坑 | seer_cleared白名单→vote 0.1 |
| 信息矛盾→排查干扰源 | drunk_warning+recluse_warning |
| 外来者对不上→推导男爵 | outsider_hint+expected_outsider_info |
| 对跳→优先排查 | claim_conflicts→vote 0.85 |
| 多源指向→证据链 | info_overlap→vote 0.5+count×0.2 |

### 邪恶推理链

| 战术 | 实现 |
|------|------|
| 信息位识别 | 首夜间谍全知→spy_shared广播 |
| 刀人优先级 | 占卜>调查>共情>功能>普通 |
| 伪装分层 | 1强信息+1弱功能+1镇民 |
| SW保险 | 存活时恶魔aggression+0.3 |
| 终局隐身 | 未提名→+0.15/回合 |

### 推理校验机制

| 校验 | 位置 |
|------|------|
| 占卜查验硬锁 | chosen0/chosen1直接赋值绕过validate |
| 调查员信息修复 | kwargs.update提到if/else外 |
| 身份一致性 | public_claims全局强制锁定 |
| 假信息数字校验 | 厨师/共情数字只能不变或+1 |
