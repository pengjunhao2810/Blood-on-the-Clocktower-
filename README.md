## 游戏规则

《暗流涌动》(Trouble Brewing) 是《血染钟楼》的入门剧本，适合 5-12 名玩家。

**阵营：**
- **善良阵营（镇民 + 外来者）**：找出并处决恶魔
- **邪恶阵营（恶魔 + 爪牙）**：存活到最后，存活 ≤2 人即获胜

**核心流程：**
1. **天黑闭眼** — 各角色按顺序使用技能
2. **私聊阶段** — 玩家私下交换信息
3. **公聊阶段** — 全体讨论
4. **提名投票** — 提名处决嫌疑人

**12 人标准配置：** 7 镇民 + 2 外来者 + 2 爪牙 + 1 恶魔

**胜负条件：**
- 善良获胜：恶魔全部死亡且无继承单位
- 邪恶获胜：存活玩家 ≤2 人且恶魔在场
- 镇长特殊：仅剩 3 人存活且无人被处决 → 直接善良获胜

## V3 核心改进

### 规则硬约束（共 16 项）

| 约束 | 效果 |
|------|------|
| 圣徒处决拦截 | 提名池排除圣徒、投票自动返回 0 |
| SW 继承限制 | 仅红唇女郎可继承，删除 `elif minions` 兜底 |
| 恶魔禁刀队友 | `valid_targets` 排除所有爪牙+其他恶魔 |
| 贞洁者规则修正 | 提名者死后跳过投票环节 |
| 猎手禁射圣徒/死者 | exclude 集合包含圣徒+已死亡玩家 |
| 胜负后冻结 | 5 个阶段入口检查 `game_record.result` |
| 双恶魔检测 | 开局统计+降级+`_evil_plan` 移除 |
| SW 存活检查 | `dead_players` 二次确认，已有恶魔则跳过 |
| 自杀传位最小天数 | 低怀疑 ≥3 天，高怀疑 ≥2 天 |
| 空泛指控过滤 | 无证据 → vote×0.4，首日再×0.5 |
| 首日保护位 | 信息位+僧侣+士兵+贞洁者 → vote 0.05 |
| 镇长终局自保 | 3人时自己票 return 0 |
| 占卜/调查查杀 | 提名命中技能范围 → +0.3 |
| 反推好人 | 提名 seer_cleared → -0.2 |
| 好人全灭允许刀队友 | `exclude_names` 仅在好人存活时生效 |
| 管家强制跟票 | 主人投票后管家复制结果 |

### 信息交叉系统

| 机制 | 实现 |
|------|------|
| `seer_cleared` | 占卜"无恶魔"→两人全局白名单→投票 0.1 |
| `info_overlap` | 多信息位指向同一人→vote 0.5+count×0.2 |
| `info_credibility` | 处决验证→信息位可信度 ±0.15(0.5~1.5) |
| 厨师位置推理 | 相邻邪恶 0→邻居各-10 嫌疑；≥1→各+5 |
| 死后复盘 | 处决好人→升投赞成者嫌疑；处决坏人→洗清赞成者 |
| 潜意识线索 | 改口身份→全局 suspicion +15 |

### 过程奖励训练

**28 个战术模式（从 19 个 Bilibili 视频提取）**

| 类别 | 模式数 | 代表性战术 |
|------|-------|-----------|
| 好人 | 9 | 信息交叉验证、装弱钓鱼、中毒推导、酒鬼信念摇摆 |
| 邪恶 | 10 | SW 保险、分层伪装、间谍选装、终局隐身、恶魔欺骗、独狼翻盘 |
| 综艺 | 4 | 保恶魔、自刀伪僧侣、自证反曝、踩人洗白 |
| 错误模式 | 6 | 时间穿越、角色浪费、反推好人、空泛指控、忽略线索、过度自证 |

18 项过程奖励信号注入：
- 刀中信息位 +0.3 | 刀队友 -0.5
- 提名对跳 +0.15 | 处决对跳 +0.25
- 处决 seer_cleared -0.4
- 交叉验证 +0.15 | 成功偏转 +0.2
- 终局隐身 +0.15/回合

### 导出增强

- 公聊：深青色文字 + 深绿色粗体人名
- 私聊：深紫色文字 + 深红色粗体人名
- 角色表：新增"原身份/备注"列，SW 继承标注"原角色: 红唇女郎"
- 私聊：每天独立键值 `D{day}_A↔B`，不覆盖
- 夜晚信息：含 SW 转换事件

### 邪恶公聊语气

从"检察官"改为"观察者"：
- 旧："你根本就是在编！今天必须出他！"
- 新："有几个细节让我在意……建议今天至少让他解释清楚。"

## AI 对抗能力

### 邪恶方已稳定掌握

- ✅ 3 套完整开局战术包（信息位对跳/硬身份扛推/机制利用）
- ✅ 分层伪装（1 强信息+1 弱功能+1 镇民）
- ✅ 爪牙护主（投票反对 + 自动转移提名）
- ✅ SW 保险思维（存活时恶魔发言更激进）
- ✅ 自刀传位（≥2 天存活+高怀疑才触发）
- ✅ 假信息闭环（不发金水、数字一致性校验）
- ✅ 终局隐身（未被提名奖励 +0.15）
- ✅ 伪装配配置校验（0 外来者局不跳外来者身份）

### 好人方已稳定掌握

- ✅ 信息交叉投票（多源指向→自动加权）
- ✅ 对跳优先处决（claim_conflicts→vote 0.85）
- ✅ 首日不投信息位/保护位（vote 0.05）
- ✅ 空泛指控过滤（无证据→vote×0.2）
- ✅ 死后复盘（自动调整嫌疑/信任）
- ✅ 厨师位置推理（相邻邪恶数→调整邻居嫌疑）
- ✅ 占卜排人白名单（seer_cleared→vote 0.1）
- ✅ 功能位隐藏（送葬/守鸦/士兵首日 40%装镇民）

### 已知瓶颈

- 12 人局稳定性偶有异常
- 好人间信息整合缺乏队长级决策能力
- 真假信息判断仍依赖单点而非全局
