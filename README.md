## 游戏规则

《暗流涌动》是《血染钟楼》的入门剧本，适合 5-12 名玩家。

### 阵营

| 阵营 | 角色类型 | 目标 |
|------|---------|------|
| 🟢 善良阵营 | 镇民 (Townsfolk) + 外来者 (Outsider) | 找出并处决所有恶魔 |
| 🔴 邪恶阵营 | 恶魔 (Demon) + 爪牙 (Minion) | 存活到最后（≤2人存活） |
| 🍺 酒鬼 | 自以为是镇民 | 实际是外来者，技能信息为假 |

### 各人数配置

| 玩家数 | 镇民 | 外来者 | 爪牙 | 恶魔 | 男爵局外来者 |
|-------|------|--------|------|------|------------|
| 5 | 3 | 0 | 1 | 1 | 2 |
| 6 | 3 | 1 | 1 | 1 | 3 |
| 7 | 5 | 0 | 1 | 1 | 2 |
| 8 | 5 | 0 | 2 | 1 | 2 |
| 9 | 5 | 2 | 1 | 1 | 4 |
| 10 | 7 | 0 | 2 | 1 | 2 |
| 11 | 7 | 1 | 2 | 1 | 3 |
| 12 | 7 | 2 | 2 | 1 | 4 |

### 核心流程

```
🌙 天黑闭眼 → 🤫 私聊 → 💬 公聊 → 🗳️ 提名投票 → ⚔️ 处决/平安日 → 🌙 天黑...
```

### 胜负条件

| 条件 | 胜方 | 触发时机 |
|------|------|---------|
| 所有恶魔死亡且无继承 | 善良阵营 | 恶魔死亡后检查 |
| 存活玩家 ≤2 人且恶魔在场 | 邪恶阵营 | 每轮结束时 |
| 仅剩 3 人存活且无人被处决 | 善良阵营 | 白天结束时（镇长技能） |
| 圣徒被处决 | 邪恶阵营 | 投票处决圣徒时立即触发 |

### 关键机制

- **红唇女郎 (Scarlet Woman)**: 场上唯一可继承恶魔身份的角色。恶魔死亡时若红唇女郎存活且总人数≥5，红唇女郎变为新恶魔。
- **男爵 (Baron)**: 增加 2 名外来者，且必有酒鬼在场。
- **酒鬼 (Drunk)**: 自以为是镇民，但所有技能信息由说书人随机给出错误结果。
- **陌客 (Recluse)**: 被信息技能查验时会错误显示为邪恶阵营。
- **占卜师红鲱鱼**: 随机一名非恶魔玩家始终被占卜师当作恶魔。
- **士兵 (Soldier)**: 免疫恶魔夜间攻击。
- **镇长 (Mayor)**: 仅剩 3 人存活且无人被处决时，善良阵营直接获胜。
- **圣徒 (Saint)**: 若被白天处决，善良阵营立即失败。

## UI 界面说明

### 主页面 (`/`)
- **左侧日志区**: 按天显示公聊发言、提名投票、处决结果
- **右侧边栏**: 私聊气泡、说书人信息面板、夜晚顺序、角色能力表、提名记录
- **控制栏**: 玩家数滑块、「重置」「导出记录」按钮
- **推进按钮**: 逐步执行角色分配→天黑→私聊→公聊→提名投票

### 私聊标签页 (`/private_chat`)
- **5 标签页**:
  1. 💬 私聊记录 — 左右气泡式对话
  2. 📜 说书人信息 — 所有人的真实身份+技能结果
  3. 🌙 夜晚顺序 — 当前阶段的角色行动顺序
  4. 📋 角色能力 — 所有角色的技能说明
  5. 🗳️ 提名记录 — 每轮提名、投票、结果

### 导出功能 (`/export`)
- 生成 `.docx` 格式 Word 文档
- 公聊青色+绿名、私聊紫色+红名
- 角色表含「原身份/备注」列（SW 继承标注）
- 每晚信息独立记录

## V3 核心改进

### 规则硬约束（共 16 项）

| # | 约束 | 效果 |
|---|------|------|
| 1 | 圣徒处决拦截 | 提名池排除圣徒、投票自动 return 0 |
| 2 | SW 继承限制 | 仅红唇女郎可继承，删除 `elif minions` 兜底 |
| 3 | 恶魔禁刀队友 | `valid_targets` 排除所有爪牙+其他恶魔 |
| 4 | 贞洁者规则修正 | 提名者死后 `continue` 跳过投票 |
| 5 | 猎手禁射圣徒/死者 | exclude 集合包含圣徒+已死亡玩家+自己 |
| 6 | 胜负后冻结 | 5 个阶段入口 `if game_record.result: return` |
| 7 | 双恶魔检测 | 开局统计→降级→`_evil_plan` 移除 |
| 8 | SW 存活+已有恶魔检查 | `dead_players` 二次确认；已有恶魔则跳过 |
| 9 | 自杀传位最小天数 | 低怀疑 ≥3 天，高怀疑 ≥2 天 |
| 10 | 空泛指控过滤 | 无证据 → vote×0.4，首日再×0.5 |
| 11 | 首日保护位 | 信息位+僧侣+士兵+贞洁者 → vote 0.05 |
| 12 | 镇长终局自保 | 3 人存活时自己票 return 0 |
| 13 | 占卜/调查查杀 | 提名命中技能范围 → +0.3 |
| 14 | 反推好人 | 提名 seer_cleared → -0.2 |
| 15 | 好人全灭允许刀队友 | `exclude_names` 仅在好人存活时生效 |
| 16 | 管家强制跟票 | 主人投票后管家复制投票结果 |

### 信息交叉系统

| 机制 | 实现 |
|------|------|
| `seer_cleared` | 占卜"无恶魔"→全局白名单→所有好人 vote 0.1 |
| `info_overlap` | 多信息位指向同一人→vote 0.5+count×0.2 |
| `info_credibility` | 处决验证→信息位可信度 ±0.15（范围 0.5~1.5） |
| 厨师位置推理 | 相邻邪恶 0→邻居各-10 嫌疑；≥1→各+5 |
| 死后复盘 | 处决好人→升投赞成者嫌疑；处决坏人→洗清赞成者 |
| 潜意识线索 | 改口身份→全局 suspicion +15 |

### 对话系统升级

| 改动 | 效果 |
|------|------|
| 邪恶公聊语气 | "检察官"→"观察者"（"今天必须出他"→"建议至少让他解释清楚"） |
| 伪装配配置校验 | 0 外来者局禁止跳圣徒/管家/陌客/酒鬼 |
| 假信息一致性 | 厨师/共情数字不能随意跳变（只能增长或不变） |
| 酒鬼信念摇摆 | 50% 概率发言注入"可能我是酒鬼/信息可能有误" |
| 假信息不发金水 | 占卜/僧侣/送葬者假模板去掉"好人/金水"类表述 |

### 导出增强

- 公聊：深青色(#006B6B)文字 + 深绿色(#008000)粗体人名
- 私聊：深紫色(#800080)文字 + 深红色(#8B0000)粗体人名
- 角色表：7 列含"原身份/备注"——SW 继承显示"原角色: 红唇女郎"
- 私聊：每天独立键值 `D{day}_A🔄B`，不再跨天覆盖
- 夜晚信息：含 SW 转换事件"红唇女郎→小恶魔"
- 首夜信息：角色表只显示首夜结果，不含后续轮次

## 训练系统

### ML 架构

```
Observation (10 维/玩家) → MLP (128 隐层) → 3 个输出头:
├── kill_head:  对每个玩家预测击杀概率
├── nom_head:   对每个玩家预测提名概率
└── vote_head:  对当前提名预测投票概率
```

### REINFORCE + 过程奖励

```
总奖励 = 胜负奖励(±1.0) + Σ 过程奖励(各事件)
```

训练轮次建议：≥4000 局（约 3-5 分钟）

```bash
cd training
python train_ml_evil.py           # 完整训练 4000 局
python continue_train.py           # 从断点继续
```

### 训练数据 Schema

位于 `training/training_schema.py`，包含：
- 5 个完整样本（从 Bilibili 视频标注）
- 28 个战术模式（9 好人 + 10 邪恶 + 4 综艺 + 5 错误）
- 4 种训练范式说明（SFT / Reward Model / RLHF / Process Reward）

## FAQ

**Q: V3 和 V2 有什么区别？**
V3 是 V2 的独立包——把所有外部依赖（core 模块、werewolf_ai 路径）内嵌到项目中，同时增加了 16 项规则硬约束、信息交叉系统、过程奖励训练、导出增强。

**Q: 如何从 V2 升级到 V3？**
V3 是独立目录，直接克隆使用即可，不需要 V2。

**Q: ML 训练需要 GPU 吗？**
不需要。MLP 策略网络很小（128 隐藏层），CPU 训练 4000 局约 3-5 分钟。

**Q: 如何查看 AI 的"真实想法"？**
说书人信息面板（`/storyteller_info`）显示所有玩家的真实身份和技能结果。

**Q: 支持多少玩家？**
5-12 人，通过页面滑块或 `/set_players/<n>` API 设置。

**Q: 私聊对话有记录吗？**
有。私聊历史存储在 `game_record["private_chat_history"]`，导出 Word 时会包含所有私聊记录。

**Q: 为什么有时候好人会处决圣徒？**
V3 已修复：圣徒被提名前、投票前两处拦截，好人不会主动提名或投票圣徒。如果仍发生，请查看说书人信息确认该玩家是否公开声明了圣徒身份。

**Q: 已知问题有哪些？**
12 人局偶有稳定性问题；好人间信息整合仍缺乏队长级统一归票能力。

## 未来计划

### 真人玩家加入（多人联机）

**架构**:
```
玩家A(浏览器) ─┐
玩家B(浏览器) ─┤
              ├→ Flask WebSocket → GameEngine → AI玩家填充空位
玩家C(手机)  ─┤
AI玩家×N      ─┘
```

**开发路线**:
| 阶段 | 内容 | 预计 |
|------|------|------|
| P0 基础联机 | WebSocket房间制，1真人+N AI | 优先 |
| P1 混合对局 | 多人真人+AI混合，身份私密 | 次优 |
| P2 完整社交 | 好友系统、战绩记录、排行榜 | 远期 |

**计划功能**:
- 房间制：创建/加入房间，设置房间密码和最大人数
- 混合对局：任意比例的真人+AI玩家（1真人+11AI 到 12真人+0AI）
- 实时推送：WebSocket 推送对局状态（新阶段/发言/投票结果），无需手动刷新
- 身份私密：真人玩家只看自己的角色和技能信息，说书人面板仅房主可见
- 私聊系统：真人之间、真人与AI之间均可私聊，AI会根据聊天内容动态调整信任度
- 投票交互：倒计时投票（可配置30s/60s/无限），超时自动弃权
- 观战模式：不参与对局的观众可观看完整流程（含所有角色身份）

### AI 对话真人化（拟人性增强）

当前 AI 发言虽然已从"检察官模板"优化为"观察者语气"，但与人脑真实玩家的对话仍有明显差距。以下是可具体落地的改进方向：

#### 1. 记忆与关系系统

**当前**: AI 每轮独立发言，不记得之前和谁聊过、关系如何。

**改进**: 给每个 AI 维护一个 `relationship_graph`：

```python
relationship = {
    "player_A": {"trust": 75, "last_chat": "day_2", "notes": "他上次说自己是调查员但我没验证"},
    "player_B": {"trust": 30, "last_chat": "day_1", "notes": "投票一直跟风，很可疑"},
}
```

**效果**: 
- 私聊时能说"上次你跟我说你是洗衣妇，我后来查了你的信息是对的"
- 公聊时能说"我跟A和B都私聊过，A的逻辑更自洽"
- 被自己信任的人背叛时有情绪反应

#### 2. 情绪状态机

**当前**: 情绪由 `_calc_emotion()` 根据怀疑度/信任度计算，但每次发言独立。

**改进**: 情绪状态跨轮次延续：

| 事件 | 情绪变化 | 对话表现 |
|------|---------|---------|
| 队友被处决 | +愤怒, +紧迫 | 语气更急促，感叹号增多 |
| 自己查杀命中恶魔 | +自信, -怀疑 | 语气更笃定，"我早就说过" |
| 被连续两次提名 | +防御, +委屈 | "我已经解释过了，为什么还要怀疑我" |
| 终局3人残局 | +紧张, +谨慎 | 话变少，每句话都反复斟酌 |
| 好人全灭无助 | +绝望, -攻击性 | 放弃抵抗，"算了，你们投我吧" |

**实现**: `player.game_state["emotion_state"] = {"anger": 0.3, "confidence": 0.7, "fear": 0.1}`

#### 3. 发言风格个性化

**当前**: 4种性格（冲动/冷静/话痨/新手），但同一性格的发言仍高度相似。

**改进**: 每个 AI 拥有独特的"语言指纹"：

| 风格维度 | 取值范围 | 示例 |
|---------|---------|------|
| 口头禅 | 每局随机1-2个 | "说实话""不是我说""你仔细想想" |
| 句式偏好 | 反问/设问/陈述/感叹 | 反问型:"你凭什么觉得他是好人?" |
| 用词风格 | 口语/书面/半文半白 | 口语:"这人不对劲" vs 书面:"该玩家行为异常" |
| 回应速度 | 快/中/慢 | 秒回 vs 等5秒再发言(模拟思考) |

#### 4. 对话上下文连贯性

**当前**: 每轮公聊独立选择模板，前后轮次之间没有承接关系。

**改进**: 强制引用上一轮的关键信息：

```python
# 发言生成时注入上一轮摘要
last_round_summary = "上一轮A提名了B, B辩护说自己是厨师, 最终5票处决了C"
# 模板中加入承接语
"刚才A提名B的事我想再聊一下——"  # 而非直接开始新话题
```

**跨轮次引用规则**:
- 被提名过的人下一轮必须回应:"关于昨天我被提名的事..."
- 信息位连续报验必须关联:"和昨晚一样，我今天的查验结果也是..."
- 有人改口后下一轮必须有人追问:"你昨天说X，今天又说Y，怎么回事?"

#### 5. 非语言信息模拟

**当前**: AI 只有纯文字发言。

**改进**: 注入带标记的非语言信息：

| 标记 | 含义 | 实现 |
|------|------|------|
| `[停顿3秒]` | 思考/犹豫 | `<span class="pause">...</span>` CSS动画 |
| `[语气加重]` | 强调 | `<strong>` 标签 |
| `[小声]` | 私密感 | `<span class="whisper">` 小号字体 |
| `[苦笑]` / `[叹气]` | 情绪表达 | 文本插入，真人感 |

#### 6. 错误与修正

**当前**: AI 几乎不说错话，显得过于完美而不真实。

**改进**: 故意加入合理的"失误"：
- 偶尔记错他人身份（概率5%），下一轮自我纠正:"哦等等，我记错了，B跳的是共情者不是厨师"
- 偶尔承认自己不确定:"说实话我现在也很混乱，信息太多了"
- 推理链中偶尔有跳跃:"我凭直觉觉得C有问题——别问我为什么，就是一种感觉"

#### 7. 社交策略多样性

| 策略 | 表现 | 适用角色 |
|------|------|---------|
| 亲和型 | 频繁私聊、主动帮助、语气温和 | 僧侣、士兵 |
| 权威型 | 语气果断、爱归票、不轻易私聊 | 镇长、猎手 |
| 隐身型 | 发言少、跟票多、不主动互动 | 爪牙、外来者 |
| 搅局型 | 反问多、质疑一切、转移话题 | 恶魔、投毒者 |
| 理性型 | 数据分析、逻辑推导、证据链完整 | 占卜师、调查员 |
